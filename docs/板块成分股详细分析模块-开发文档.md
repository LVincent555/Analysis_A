**创建时间**: 2025-11-09

**优先级**: 高

## 📌 项目背景

### 现有功能
系统已实现板块热度加权分析功能（`/api/industry/weighted`），可以查看所有板块的排名和热度指标。但缺少对**单个板块**的深度分析能力。

### 新模块定位
本模块是**全新独立模块**，专注于单个板块的深入分析，提供：
- 查看板块包含的所有成分股
- 分析板块内部结构和特征
- 追踪板块历史趋势
- 对比多个板块的差异

### 术语说明

⚠️ **重要命名约定**:

```
术语层级：
├─ "行业" (industry) ← 代码中的术语
│   └─ 实际含义：板块分类
│   └─ 数据来源：stocks.industry 字段
│   └─ 示例：退货、食品、建材、化学、电气...
│
└─ "板块" (sector) ← 独立数据源
    └─ 实际含义：独立的板块技术指标
    └─ 数据来源：sectors.sql (有版权争议)
    └─ 示例：工程建设、社区团购、转基因...
    └─ 特点：无法追溯包含哪些股票

本模块分析的是 stocks.industry（代码中称为"行业"，实际是板块分类）
与 sectors.sql 中的独立板块数据无关
```

### 5. 多榜单交叉信号排序 ⭐⭐⭐ **创新功能**

**功能描述**:
通过分析股票在多个关键榜单中的出现情况，计算"信号强度"，优先展示多重验证的优质股票。

**核心理念**:
```
传统排序的局限：
- 单一维度容易有噪音
- 排名50的股票 vs 排名51的股票差别不大
- 无法识别"全面优秀"的股票

多榜单交叉验证：
- 同时出现在3个榜单 > 只在2个榜单 > 只在1个榜单
- 信号越多 = 确定性越高 = 优先级越高
- 降低偶然性，提高可靠性
```

**榜单定义**:
1. **最新热点榜**: 全市场排名前100/500
2. **排名跳变榜**: 排名提升幅度≥100的股票
3. **稳步上升榜**: 连续3天以上排名上升
4. **涨幅榜**: 当日涨跌幅≥5%
5. **成交量榜**: 换手率≥10%

**信号强度计算公式**:
```python
signal_strength = (
    0.30 * 热点榜信号 +      # 权重最高
    0.25 * 跳变榜信号 +
    0.25 * 稳步上升信号 +
    0.10 * 涨幅榜信号 +
    0.10 * 成交量榜信号
)

其中：
- 热点榜TOP100 = 0.30, TOP500 = 0.15
- 跳变≥200 = 0.25, 跳变≥100 = 0.15
- 上升≥5天 = 0.25, 上升≥3天 = 0.15
- 涨幅≥5% = 0.10
- 换手率≥10% = 0.10
```

**排序模式**:

**L1: 信号强度排序（推荐）**
```python
stocks.sort(key=lambda x: (
    -x.signal_count,      # 第一优先级：信号数量
    -x.signal_strength,   # 第二优先级：信号强度
    x.rank                # 第三优先级：全市场排名
))
```

**L2: 全市场排名排序（传统）**
```python
stocks.sort(key=lambda x: x.rank)
```

**L3: 总分排序（质量优先）**
```python
stocks.sort(key=lambda x: -x.total_score)
```

**API端点**: 在成分股列表API中增加`sort_mode`参数
```python
GET /api/industry/{industry_name}/stocks?sort_mode=signal
# sort_mode可选: signal | rank | score
```

**返回数据增强**:

## 📊 数据流程

### 1. 成分股查询流程
```
用户点击"食品"板块
  ↓
前端发起请求: GET /api/industry/食品/stocks
  ↓
后端检查缓存
  ↓ (缓存未命中)
从memory_cache获取指定日期数据
  ↓
筛选industry="食品"的股票
  ↓
按指定字段排序
  ↓
返回结果并缓存
  ↓
前端展示列表
```

### 2. 多榜单信号计算流程
```
用户选择"信号强度排序"
  ↓
前端发起请求: GET /api/industry/食品/stocks?sort_mode=signal
  ↓
后端从memory_cache获取数据
  ↓
获取各个榜单数据：
  - 排名跳变榜（从历史数据计算）
  - 稳步上升榜（从历史数据计算）
  - 热点榜、涨幅榜、成交量榜（从当日数据）
  ↓
为每只股票计算信号强度：
  for stock in 板块股票:
    signals = []
    score = 0.0
    if stock in 热点榜TOP100: score += 0.30
    if stock in 跳变榜: score += 0.25
    if stock in 稳步上升榜: score += 0.25
    if stock.price_change >= 5%: score += 0.10
    if stock.turnover >= 10%: score += 0.10
  ↓
按信号强度排序
  ↓
返回带信号标记的股票列表
  ↓
前端展示：信号Badge + 强度星级
```

## ⚠️ 注意事项

### 1. 性能考虑
- 单板块的成分股数量：50-300只
- 查询效率：从memory_cache直接筛选，O(n)时间复杂度
- 缓存命中率：预计>80%（同一板块被频繁查询）

### 2. 边界情况
- 板块名称不存在：返回404
- 指定日期无数据：返回最近可用日期的数据
- 板块内无股票：返回空列表（理论上不会发生）

### 3. 兼容性
- 不影响现有功能
- API路径独立，不冲突
- 前端模块独立，可选加载

## ✅ 已确定决策

### 1. 排序方式 ✅ **已确定**
采用**多榜单交叉信号排序**方案，提供3种排序模式：
- **L1 (推荐)**: 信号强度排序 - 多重验证，优先展示确定性机会
- **L2**: 全市场排名排序 - 传统视角
- **L3**: 总分排序 - 质量优先

### 2. 排序和筛选 ✅ **已确定**
支持以下排序方式，默认信号强度排序：
- 信号强度（signal_strength）⭐ 默认
- 全市场排名（rank）
- 总分（total_score）
- 涨跌幅（price_change）
- 成交额（volume）
- 市值（market_cap_billions）

### 3. 信号筛选器 ✅ **已确定**
提供多个筛选条件：
- 仅显示热点榜（前100/500）
- 仅显示跳变榜（排名提升≥100）
- 仅显示稳步上升（连续3天+）
- 仅显示涨幅>5%
- 仅显示多信号股票（≥2个信号）

### 4. 前端展示方式 ✅ **已确定**
**两级展示：对话框 + 独立页面**

**级别1：对话框（快速预览）**
- 从板块列表点击板块名称 → 弹出对话框
- 显示内容：
  - 板块基本信息（股票数、4维指标）
  - TOP 10成分股列表（信号强度排序）
  - "查看完整分析"按钮 → 跳转到独立页面
  
**级别2：独立页面（完整分析）**
- 路由：`/industry-detail/:industry_name`
- 显示内容：
  - 完整成分股列表（支持排序、筛选、分页）
  - 所有图表（排名分布、信号分布、历史趋势等）
  - 信号阈值调节器
  - 历史信号追踪
  - 板块对比功能
- 返回按钮 → 回到原页面

### 5. 信号权重策略 ✅ **已确定**
**Phase 1：固定平衡型权重**
```python
signal_strength = (
    0.30 * 热点榜信号 +      # 最重要
    0.25 * 跳变榜信号 +
    0.25 * 稳步上升信号 +
    0.10 * 涨幅信号 +
    0.10 * 成交量信号
)
```

**Phase 2（未来）：预设策略**
- 保守型、激进型等多种策略可选

### 6. 信号阈值 ✅ **已确定**
**可调节，但提供默认值**
- 热点榜阈值：TOP 100 / TOP 500（默认100）
- 跳变榜阈值：≥100 / ≥200（默认100）
- 稳步上升阈值：≥3天 / ≥5天（默认3天）
- 涨幅榜阈值：≥5%（可调节1%-10%，默认5%）
- 成交量榜阈值：换手率≥10%（可调节5%-20%，默认10%）

### 7. 历史信号追踪 ✅ **已确定**
**要实现**
- 显示股票过去7天的信号历史
- 标识：连续出现在榜单 vs 偶尔出现
- 示例：
  ```
  平安银行 - 信号历史
  📊 热点榜: ✅✅✅✅✅ (连续5天TOP100)
  📈 跳变榜: ✅❌❌✅❌ (2次跳变)
  ⬆️ 稳步上升: ✅✅✅✅❌ (曾连续4天)
  ```

### 8. 板块对比数量 ✅ **已确定**
**做成选项：2个 / 3个 / 5个**
- 默认：3个板块对比
- UI提供选择器切换
- 根据选择动态调整布局

### 9. 图表需求 ✅ **已确定**
**独立页面包含所有图表**：
1. 排名分布饼图（TOP100/500/1000/other）
2. 信号分布条形图（各榜单股票数量）
