## 核心理念：里世界与表世界 (The Dual World)

不仅仅是被动防御，而是构建一个完整的“表世界”作为蜜罐，将真实的 API 隐藏在“里世界”中。

- **表世界 (Outer World)**：
  - 一个看似充满漏洞、维护糟糕的博客系统（WordPress/PHP 风格）。
  - 实际上是一个高互动性的蜜罐（Honeypot），充满了陷阱、死循环和垃圾数据。
  - **目的**：吸引攻击者火力，消耗其资源，满足“混沌邪恶”的娱乐需求。

- **里世界 (Inner World)**：
  - 真实的股票分析系统 API。
  - **入口**：仅通过通过特定请求头（如 `X-Real-IP` 或特定的 `User-Agent`）或精确的 API 路由（`/api/v1/*`）进入。
  - **保护**：严格的鉴权和访问控制。

## 陷阱设计 ( The Traps)

### 1. 葫芦娃无限流 (The Hulu Stream)
- **触发路径**：`/backup.sql`, `/db_dump.tar.gz`, `/.git/config`, `/admin/export`
- **行为**：
  - 返回 `200 OK`，`Content-Type: application/octet-stream`。
  - **机制**：通过流式响应（StreamingResponse）持续发送随机的“葫芦娃”台词、无意义的 SQL 插入语句（`INSERT INTO hulu_brothers VALUES...`）以及二进制垃圾。
  - **效果**：攻击者的爬虫/下载工具会以为发现了大文件而持续下载，直到填满其硬盘或超时。
  - **控制**：限制传输速率（如 10KB/s），避免反向消耗服务器流量。

### 2. 时间停滞 (The Tarpit)
- **触发路径**：`/wp-login.php`, `/admin/login`, `/manager/html`
- **行为**：
  - 接受连接，先发送 HTML 头部，然后每隔 5-10 秒发送一个 HTML 注释字符（`<!-- loading -->`）。
  - **效果**：占用攻击者的连接线程，使其扫描效率极具下降，且难以判断请求是否结束。

### 3. Gzip 炸弹 (The Zip Bomb - Safe Mode)
- **触发路径**：`/assets/main.bundle.js`
- **行为**：
  - 返回一个高压缩比的 gzip 流（如 10MB 的全 0 数据，压缩后仅几 KB）。
  - **效果**：浏览器或自动解压的爬虫在展开数据时会消耗大量内存。
  - **注意**：控制解压后的大小在 100MB 以内，避免误伤自己或触发云服务商的防御机制。

### 4. 登录黑洞 (The Login Void)
- **触发路径**：POST `/wp-login.php`
- **行为**：
  - 接收 POST 数据。
  - 随机等待 5-20 秒（模拟数据库慢查询）。
  - 永远返回 `200 OK` 但内容是“密码错误”的页面。
  - **效果**：让攻击者无法区分是网络慢、数据库慢还是被拦截，从而浪费时间在爆破上。

## 架构设计

在 FastAPI 中通过 Middleware 实现分流：

```python
async def chaos_gate_middleware(request: Request, call_next):
    # 1. 判断是否进入里世界（真实API）
    # 条件：访问 /api/ 开头
    if is_inner_world(request):
        return await call_next(request)
    
    # 2. 否则进入表世界（混沌陷阱）
    # 路由到伪造的 Router 或直接返回假博客首页
    # 404 请求也会被拦截并返回假博客
    return await chaos_router_dispatch(request)
```

## 风险控制

> [!WARNING]
> **切勿玩火自焚**
> 1. **带宽限制**：必须限制蜜罐响应的速率。
> 2. **连接限制**：设置最大并发连接数（如 50），超过则直接断开，防止耗尽服务器资源。
> 3. **日志隔离**：蜜罐日志必须写入单独的 logger（`chaos.log`），严禁混入业务日志。
