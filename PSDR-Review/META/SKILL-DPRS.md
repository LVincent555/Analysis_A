# SKILL: DPRS 决策流程（PRB → SUG → DEC → RES）

> **适用场景**：发现问题或风险，需要规范化跟踪、决策与复盘。可复用到任意项目。

---

## 0. 总览

- **PRB**: Problem Brief / Root-cause 假设。记录事实、影响、初步假设，不做方案。
- **SUG**: Suggestions。提出备选方案，拆解步骤、成本、风险。
- **DEC**: Decision。明确选择的方案/范围/责任/时间。
- **RES**: Result。执行后记录结论、回归结果、残余风险/后续项。

**子案/后缀命名**：
- 同一问题多方案：`SUG-003-A-xxx` 或 `SUG-003-001-xxx`
- 外部引用问题：`PRB-引用-XXX`
- 引用相关建议：`SUG-引用-XXX`

**顶层使用场景**：
- 跨模块的架构决策
- 多模块共享的技术结论

**模块级使用场景**：
- 模块内部问题
- 具体实现方案
- 局部优化决策

## 2. PRB（问题陈述）

必备要素：
- **现象/事实**：日志、指标、复现步骤。
- **影响**：严重度、范围。
- **初步假设**：可能原因（允许多条），不写方案。
- **阻塞/紧急度**：是否影响上线/截止日期。

**文件头模板**：
```markdown
**日期**: YYYY-MM-DD
**状态**: [OPEN|SOLVED|DEFERRED|OBSERVED|OBSOLETE] {标签1}{标签2}...
**类型**: Theory / Writing / Experiment / Infra / ...
**层级**: 顶层 | 模块-XXX
**关联**: SUG-YYY, DEC-ZZZ
```

**文档模板**：
```markdown
# PRB-XXX: 简洁标题

**日期**: / **状态**: / **类型**: / **层级**: / **关联**:

## 1. 现象
（观察到的事实）

## 2. 影响
（严重度、范围）

## 3. 初步假设
（可能原因，不写方案）

## 4. 下一步
（需要收集的证据）
```

---

## 3. SUG（方案建议）

目标：提出可选方案 + 步骤 + 风险，供决策用。

- **选项列表**：每个选项包含：做法、预期收益、成本/风险、验证方式。
- **推荐**：如有倾向可说明理由。
- **不执行的方案**：可标明放弃原因，避免反复讨论。

**文档模板**：
```markdown
# SUG-XXX: 简洁标题

**日期**: / **状态**: / **类型**: / **层级**: / **关联**:

## 选项 A：方案名称
- **做法**：
- **收益**：
- **风险**：
- **验证**：

## 选项 B：...

## 推荐
X（理由）

## 前置条件/依赖
```

---

## 4. DEC（决策）

目标：锁定方案与范围，明确责任与时间。

**文档模板**：
```markdown
# DEC-XXX: 简洁标题

**日期**: / **状态**: / **类型**: / **层级**: / **关联**:

## 决策
采用 SUG-XXX 的选项 Y

## 范围
- 做：
- 不做：

## 负责人 / 截止时间

## 验收标准
（指标/测试/文档）
```

---

## 5. RES（结果/复盘）

目标：记录执行结果、回归、遗留风险。

**文档模板**：
```markdown
# RES-XXX: 简洁标题

**日期**: / **状态**: / **类型**: / **层级**: / **关联**:

## 执行摘要
（做了哪些改动/实验）

## 结果与证据
（指标/日志/截图）

## 是否满足验收
是/否（原因）

## 遗留风险 / 后续行动
```

---

## 6. 流程原则

1. **状态透明**：文件头部注明状态（OPEN/SOLVED/DEFERRED 等）。
2. **层级明确**：标注"顶层"或"模块-XXX"，避免混淆。
3. **单向流转**：PRB → SUG → DEC → RES；若方案失败，可回到 SUG 补充新选项。
4. **小步快迭代**：SUG/DEC/RES 尽量短平快，避免长时间停留在"开放问题"。
5. **证据优先**：PRB 记录事实；RES 附指标/日志，避免口述结论。

## 7. 可选附加

### PRB-引用（外部咨询）
当需要外部知识/搜索时，先写 PRB-引用，整理问题与 prompt，再将外部回答转化为 SUG。
详见 [规范-PRB-引用提问流.md](./规范-PRB-引用提问流.md)（如有）。

### 状态枚举
`[OPEN|SOLVED|DEFERRED|OBSERVED|OBSOLETE|ACTIVE]`

### 推荐标签集
`{引用}{实验}{架构}{理论}{写作}{审稿}{数据}{自动化}{研究方向}`

### 关联链路
在文件头增加 `关联:` 字段，列出上/下游文档（PRB/SUG/DEC/RES），形成可追踪链。

### 索引维护
- 顶层索引：`/review/META/INDEX.md`
- 模块级索引：各模块自行维护（可选）

### Temp 文件
`SUG/temp-*`、`PRB/temp-*` 用于临时想法/引用，不纳入正式编号。
